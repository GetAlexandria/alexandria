# Alexandria

Documentation system giving AI agents implicit product context for aligned decisions during development.

## The Core Idea

**If you'll need to say it again, put it in the library.**

Without specification context, agents produce technically correct but contextually wrong outputs. This library makes implicit knowledge explicit and queryable.

## Quick Start

### For AI Agents

1. Read **reference.md** — type taxonomy, templates, naming, conformance
2. Before touching a feature, find its card and follow the wikilinks
3. After completing work, update affected cards' WHEN sections

### For Humans

1. Open this folder in **Obsidian** for graph view and wikilink resolution
2. Review AI contributions to the library
3. Resolve ambiguities agents flag with `**HUMAN JUDGMENT NEEDED:**`

## Library Structure

### Rationale (`/rationale/`) — WHY we build

| Type       | Purpose                                      |
| ---------- | -------------------------------------------- |
| Product Thesis | Guiding philosophies — the bets we're making |
| Principle  | Judgment guidance — rules of thumb           |
| Standard   | Testable specs — concrete rules              |

### Product (`/product/`) — WHAT gets built

| Type       | Purpose                              |
| ---------- | ------------------------------------ |
| Domain     | Top-level workspaces                 |
| Section    | Nested spaces within domains         |
| Governance | Cross-domain persistent elements     |
| Template   | Spatial/visual fabric                |
| Component  | Specific UI widgets                  |
| Artifact   | Content objects users create/edit    |
| Capability | Actions/workflows users perform      |
| Primitive  | Core data entities                   |
| System     | Invisible mechanisms                 |
| Agent      | AI team members                      |

### Experience (`/experience/`) — HOW it feels

| Type      | Purpose                           |
| --------- | --------------------------------- |
| Loop            | Repeating engagement cycles       |
| Journey         | Multi-phase progression arcs      |
| Experience Goal | Target emotional experiences      |
| Force           | Emergent cross-system behaviors   |

### Sources (`/sources/`)

Frozen snapshot documents that informed card creation. Not searchable by agents.

## Card Anatomy

Every card has 5 dimensions: WHAT, WHERE, WHY, WHEN, HOW.

See `reference.md` for templates and naming conventions.
