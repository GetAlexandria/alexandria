# Demo Video

Use this skill when coordinating the full demo-video process for Alexandria.

1. Read `references/alexandria-demo-video.md` for the default framing.
2. Use `demo-thesis` first when the audience, belief change, proof burden, or
   CTA is not already locked.
3. Use `story-spine` next to create hook options, causal beats, and a
   claim-proof map.
4. Use `demo-path` next to create exact product steps, required data, risks, and
   fallback paths.
5. Only then move into later stations: product prep, capture plan, recording,
   edit, finish, derivatives, launch, and direct outreach.
6. Flag unsupported claims, missing proof, fragile product states, and workflow
   gaps instead of smoothing over them.
